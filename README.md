# Japan Medical Area Mapper 🗺️

日本の二次医療圏単位のデータをインタラクティブに可視化し、美しいヒートマップ（コロプレスマップ）を作成・ダウンロードできるWebアプリケーションです。

Pythonの強力な地理空間分析ライブラリ `GeoPandas` と、美しいUIを簡単に構築できる `Streamlit` を組み合わせて構築されています。

## ✨ 主な機能

* 📂 **簡単データアップロード**: 二次医療圏コードを含むCSVまたはExcelファイルをアップロードするだけで自動マッピング。
* 📆 **年度別区割りの切り替え**: 
  * 2024年以降の新区分（秋田県の医療圏変更、浜松市の行政区再編を反映）
  * 2023年以前の旧区分
* 🎯 **都道府県ズーム（オートフォーカス）**: 特定の都道府県のみを選択し、その地域にぴったりズームインした拡大マップを100%ズレのない高精細境界線で描画。
* 🎨 **柔軟なプロットオプション**:
  * **連続変数（グラデーション）**: カラーマップの変更（OrRd, Blues, viridis等）や、最小値・最大値の境界値スライダーによる範囲調整。
  * **カテゴリカル変数（離散色）**: カテゴリダミー変数（例：0, 1, 2...）をワンクリックで個別色分け。
* 🗺️ **プロフェッショナルな地図デザイン**:
  * 都道府県境界線はしっかりとした黒太線で強調、県内部の二次医療圏境界は繊細なグレー細線で均一に配置。
  * 沖縄県は「全国」表示時のみ自動で左上にすっきりとインセット配置（個別表示時は元の正しい緯度経度でズーム）。
* 💾 **超高解像度ダウンロード**: Retinaディスプレイや印刷にも耐える **DPI 300** の極めてクッキリとした美しい画像を保存可能。

## 🚀 ローカル環境での起動方法

ご自身のPC（Mac/Windows）でアプリを動かす手順です。

### 1. 依存パッケージのインストール
ターミナルで本リポジトリのディレクトリに移動し、必要なライブラリをインストールします。

```bash
pip install -r requirements.txt
```

### 2. アプリの起動
以下のコマンドで Streamlit サーバーをローカルで起動します。

```bash
streamlit run app.py
```
起動すると、ブラウザが自動的に立ち上がり `http://localhost:8501` でアプリが利用可能になります。

## ☁️ Webへの無料一般公開手順（Streamlit Community Cloud）

GitHub経由でこのアプリを無料公開し、誰でもブラウザから使えるURLを発行する手順です。

1. **GitHubへアップロード**:
   本リポジトリをあなたのGitHubアカウントにアップロード（Push）します。
2. **Streamlit Cloudへサインイン**:
   [Streamlit Community Cloud](https://share.streamlit.io/) にアクセスし、GitHubアカウントでログインします。
3. **デプロイ（公開）設定**:
   「New app」ボタンを押し、以下の通り指定します。
   * **Repository**: あなたのGitHubリポジトリ名（例：`username/japan-medical-mapper`）
   * **Branch**: `main`
   * **Main file path**: `app.py`
4. **公開完了**:
   「Deploy!」ボタンを押すと、数分で自動構築され世界中に一般公開されます！

## 📊 アップロードするデータの推奨フォーマット

アプリにアップロードするCSVやExcelは、以下の構成になっている必要があります。

| 医療圏コード | 医師偏在指標 | ... |
| :--- | :--- | :--- |
| 0101 | 94.3 | ... |
| 0102 | 51.2 | ... |
| 1301 | 185.4 | ... |

* **医療圏コード列**: 4桁の文字列（例：`0101`）を推奨します。数値型であっても、アプリ側で自動的にゼロ埋め4桁に補正（`101` ➡ `0101`）して安全に照合します。
* 列名の自動検出機能を備えているため、コード列の名称は「医療圏コード」「二次医療圏コード」「code」など「コード」という文字が入っていれば自動選択されます（手動での列選択も可能です）。
